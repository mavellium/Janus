analyzeSeoUrl.ts — Server Action com fluxo Zod → auth → guard de empresa → rate limit (20/dia por empresa) → fetch → parse (cheerio, dependência nova) → score → persistência no novo model SeoAnalysis (migration aplicada à mão via migrate deploy, porque o shadow database do migrate dev quebra numa migration antiga pré-existente de guest_posts — vale corrigir esse histórico depois).
